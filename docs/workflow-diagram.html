Demo Talking Points
On the pipeline
One stage, one job
"Each agent has a single focused prompt. The intake agent doesn't plan. The planner doesn't write tests. That separation makes it easier to debug, swap, and explain — and mirrors how you'd build a real internal tool."
On observability
The AgentOps layer
"Every stage is timestamped in trace.json — start time, duration, retry count, status. In production that feeds a monitoring dashboard. Here it's a JSON file, but the discipline is identical: you can't improve what you can't observe."
On relevance
Connecting to Relevate
"This uses a fictional hypertension brief, but the workflow maps directly to a real technical brief — parse it, plan it, build the artifact, QA it, flag the risks, score it. The input changes. The discipline doesn't."
